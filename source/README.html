[01] About

TkWiCe is a high featured, easy to use, multilingual, free and open source software application for wine cellar management. It provides a fast overview of your wine bottles.

The software is written in native Tcl/Tk, so it works on all known operating systems (like Windows, Linux, Unix, Mac OS X etc.).

[02] Development Maximes

  • Focus is on pleasure, not on science.
  • Function conquers eyecandy.
  • The database is kept in readable text files.
  • Fits on displays with 1024x768 pixel.
  • Old computers are new enough.
  • Works platform independent.
  • Free - means liberty, free of charge and open source.

[03] Requirements

  • software: Tcl/Tk 8.5 or newer
  • cpu: 233 MHz will be slow but work, 1.4 GHz are enough and to spare
  • screen: 1024x786 pixel at 16 bit
  • program diskspace: 2.1 MB
  • database diskspace: depends on usage (expect 1 MB each 50 wines)
  • memory: needed (allocates up to 35 MB)
  • wine: one bottle (recommended: more)

TkWiCe is mostly programmed on a machine running FreeBSD. It is so far known to run on any Windows, Mac OS X (ugly), Linux or FreeBSD operating systems.

A UTF8 encoding file system is recommended - otherwise a few rare characters you may add to the database cannot be written correctly to the hard disk (but: TkWiCe will remain fully applicable).

[04] Installation

There is a install wizard ("setup.tcl") shipped with this softare - use it. Otherwise just copy the program directory somewhere you want it, and execute "tkwice.tcl" (in fact there is no installation necessary). If you need advice in a installation by hand, there is a manual in the main directory of the software archive.

[05] Possibilities & Limitations

TkWiCe is not designed to be the fastest database - instead every wine gets an own plain text file with everything in it (the photos are of course separated - they aren't plain text). The advantages: You can build up own scripts to complete TkWiCe with functions you are missing, and as the content can be read out even without TkWiCe, everything is as "future-sure" as possible.

As a result of own plain text files the maximum amount of datasets is a) limited to the filesystem you are using, and b) to 999.999 different wines. That's every day another wine in the next 2.740 years - I think this should be enough (if it's not and you need a real database behind, it shouldn't be a problem for you to pay me for such an interface). Also the maximum amount of bottles of each wine is 999. To solve this "problem" you also could pay me - six really great bottles shouldn't be a problem for someone with such a wine cellar ;)
