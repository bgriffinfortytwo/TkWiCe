Installation Notes

The official software archive is shipped with a install wizard named »setup.tcl« (if you want or need to install manually instead, there is a manual at the bottom of this file). But:

  • The whole software package requires the Tcl/Tk interpreter on your computer.
  • The installation may fail when executed from a write only filesystem (like a compact disc). Copy everything first on your harddrive.

Executing Tcl programs is quite simple then - basic syntax: »interpreter scriptname«. The interpreter name is the executable from your Tcl/Tk installation »tclsh« (name often followed by version number) - or former »wish«, which may still be the the only choice on some installations. The script names in this package are »setup.tcl« for the install wizard, and »tkwice.tcl« for TkWiCe. Example:

/path/to/tclsh8.5 /path/to/setup.tcl

But: Most users mustn't know anything about how to execute Tcl scripts. Just executing »setup.tcl« or »tkwice.tcl« will call the interpreter automatically - and if not, most desktops will ask with what to open such »*.tcl« files in future by default.

Tcl/Tk

To run TkWice - and even to execute »setup.tcl« - you need Tcl/Tk (recommended version 8.5 - 8.4 should work, too) already installed on your computer. On Unix / Linux it should be shipped with your operating system (in fact only Windows users need to get Tcl/Tk extra for sure); Otherwise you can get it for example from:

http://www.activestate.com/

Choose on the website "Language Distributions -- ActiveTcl -- Get ActiveTCL - Download". Note that the register formular there is optional. It's free software. Also note that the binaries from www.activestate.com aren't artists of file sizes, but they work. Alternative you may get a (binary) Tcl/Tk distribution from:

http://www.evolane.com/software/etcl/ (works)
http://wintcltk.berlios.de/ (untested)

If you're a computer expert you may want source to compile instead of the above binaries; In that case visit http://www.tcl.tk/, the home of Tcl/Tk. Their separate Tcl and Tk package is much smaller - and will work, too. Indeed this is absolutely unusable for average users. And you have for sure to take into account the graphic extensions manually, too.

Img Extension

Note: Img is not absolutely necessary to use TkWiCe!

And: You can ignore this chapter, if

  • you do not intend to add photos of your wine bottles to the database,
  • or your Tcl/Tk distribution is from www.activestate.com,
  • or the install wizard doesn't complain about a missing Img extension.

In any other cases please take sure that the »Img« extension to Tcl/Tk is installed. Most users may get it automatically with installing Tcl/Tk, but if not:

http://sourceforge.net/projects/tkimg/

On unix-like distributions there should be a package named libimg, imglib, tklib, libtk, tkimg, imgtk or similar providing this software. It is nearly unthinkable that you've got to install it on your own. And you can add this extension anytime after installing TkWiCe - your wine cellar software will use it as soon as found. Note that this extension is no dependency for the photo feature - it is optional. But if you use the photo feature without Img, your database size will explode (factor 6), your images will be reduced to a lower color palette, and speed will slow down. But everything works without it, too.

ImageMagick

Note: ImageMagick is not absolutely necessary to use TkWiCe!

By default TkWiCe crops photos to 500x300 pixel. On unix operating systems the ImageMagick tools are used to scale them to the right size first, but of course this will only be done if installed. Most distributions will provide ImageMagick, but if you need to get it elsewhere:

http://www.imagemagick.org/

On Windows and Macintosh operating systems this feature isn't supported just because of the impossible test environment of the author ...

Deinstallation

In fact the install wizard performs only some stupid copy actions, so you can safely delete the software again. (If you don't know anymore where the corresponding files are, just start the program and look to the bottom lines of the window - it tells you where the program, and where the database is stored. If that is somehow impossible, scan your disk for the program name ...)

This software won't make or leave any entry in a registry or similar; None conscientiously written end-user software (!) needs such questionably things.

Installation By Hand

Everything from here on is only relevant if you do not use the install wizard! Nevertheless, Tcl/Tk is of course still needed.

Move Directory

You ca run TkWiCe right where it is, but in most cases you may want to move the program directory »source« somewhere you want it. By doing this you could rename it, too. For example:

  • Unix/Linux/BSD: /usr/local/share/tkwice
  • Windows: C:\Program files\tkwice
  • Macintosh: /Applications/tkwice

If your in front of a operating system with file permissions, check them. If that sounds strange to you, you might not work on such a system (or there is no need for a user separation on your computer) - forget it.

Run It

Execute »tkwice.tcl«; The syntax may differ from operating system and installation ... some examples:

  • tkwice.tcl
  • ./tkwice.tcl
  • tclsh tkwice.tcl
  • tclsh8.5 tkwice.tcl
  • tclsh /this/directory/tkwice.tcl
  • /path/to/tclsh8.5 /this/directory/tkwice.tcl

(On some installations you may use the former »wish« instead of the newly recommended »tclsh«.)

On most systems a double mouse click on »tkwice.tcl« will exectute the right syntax. The basic thing behind the call is to start the interpreter "tclsh" by telling about the script »tkwice.tcl«. If this works you might want a little more comfort:

Comfort

Optionally:

  • On Unix/Linux/BSD you may add a symlink to get »tkwice.tcl« in the $PATH, so you're able to startup from everywhere:
    ln -s /usr/local/bin/tkwice /this/directory/tkwice.tcl
  • On Windows you may add a shortcut (not a copy!) from »tkwice.tcl« to your desktop (can be done for example by right clicking on »tkwice.tcl«).
  • On Macintosh I've got no clue.

Optionally you can define an icon. There are several included:

  • Sizes: 16/16, 24/24, 32/32, 48/48 and 64/64 pixel
  • Format: ico, xpm and png

You've got to find out yourself which one you need - there are desktops for all of this, and there is no way to set this up for you automatical. As a proposal:

  • Windows: tkwice.ico
  • KDE / GNOME : tkwice.png
  • Window Maker: ico/tkwice48.png

The usual 32/32 pixel images are in the main directory, all others you will find in the programs subdirectory named »ico«.
